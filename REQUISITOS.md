# Requisitos do Sistema de Almoxarifado
## Requisitos Funcionais
São as funções e ações que o sistema deve realizar.

1. Exemplos:
    Fazer login de usuário.
    Cadastrar clientes.
    Buscar personagens em uma API.
    Exibir informações na tela.
    Permitir editar ou excluir dados.
    Gerar relatórios.
    Enviar mensagens ou notificações.

2. Características:
    Descrevem o que o sistema faz.
    Estão ligados às funcionalidades principais.
    Normalmente são visíveis para o usuário.

## Requisitos não Funcionais
São as regras de qualidade e desempenho do sistema.

3. Exemplos:
    O sistema deve carregar em até 2 segundos.
    A aplicação deve funcionar em celulares e computadores.
    Os dados devem ser protegidos com senha.
    O sistema deve ter disponibilidade 24 horas.
    A interface deve ser fácil de usar.
    A API deve suportar muitos usuários ao mesmo tempo.

 4. Características:
    Descrevem como o sistema deve funcionar.
    Relacionados à qualidade, segurança e desempenho.
    Nem sempre são percebidos diretamente pelo usuário.
